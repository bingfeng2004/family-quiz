# 家庭知识竞答应用 - 功能详细设计

## 一、游戏核心流程

### 1.1 游戏状态机

```
[等待开始] → [题目展示] → [抢答阶段] → [答题阶段] → [结算展示] → [下一题/结束]
     ↑           ↓            ↓            ↓            ↓
     └───────────┴────────────┴────────────┴────────────┘
```

### 1.2 详细流程

| 阶段 | 玩家操作 | 系统响应 | 时间控制 |
|------|----------|----------|----------|
| 等待开始 | 输入昵称、选择主题 | 展示房间信息、参与者列表 | 无限制 |
| 题目展示 | 无 | 显示题目和选项、开始倒计时 | 1s |
| 抢答阶段 | 点击"抢答"按钮 | 第一个抢到的玩家获得答题权 | 0-倒计时结束 |
| 答题阶段 | 选择答案并提交 | 判断对错、计算分数、展示解析 | 15s |
| 结算展示 | 无 | 显示本题结果、当前排名 | 3s |
| 游戏结束 | 无 | 显示最终排行榜、颁发奖励 | 无限制 |

## 二、核心功能模块

### 2.1 用户模块

#### 2.1.1 用户身份
- **游客模式**：输入昵称即可参与，无需注册
- **登录模式**：微信一键登录，保留历史数据

#### 2.1.2 用户数据结构
```javascript
User {
  id: string,           // 唯一标识
  nickname: string,     // 昵称（2-10字符）
  avatar: string,       // 头像URL
  totalScore: number,   // 历史累计积分
  gameCount: number,     // 参与游戏次数
  correctRate: number,  // 历史正确率
  badges: string[],     // 获得的徽章
  createdAt: timestamp
}
```

### 2.2 房间模块

#### 2.2.1 房间状态
```
[空闲] → [等待中] → [进行中] → [已结束]
```

#### 2.2.2 房间数据结构
```javascript
Room {
  id: string,              // 房间ID（6位数字）
  hostId: string,          // 房主ID
  status: 'waiting' | 'playing' | 'ended',
  theme: string,           // 答题主题
  settings: {
    questionCount: number, // 题目数量（默认10）
    answerTime: number,    // 答题时间（默认15秒）
    scoreRule: {           // 积分规则
      correct: +1,
      wrong: -1
    }
  },
  players: Player[],       // 当前玩家列表
  currentQuestion: number, // 当前题目索引
  questions: string[],     // 题目ID列表
  createdAt: timestamp
}
```

#### 2.2.3 玩家数据结构
```javascript
Player {
  id: string,
  nickname: string,
  avatar: string,
  score: number,          // 当前分数
  status: 'waiting' | 'ready' | 'answered',
  answerTime: number,     // 抢答时间戳
  isHost: boolean
}
```

### 2.3 题库模块

#### 2.3.1 题目类型
| 类型 | 说明 | 答案格式 |
|------|------|----------|
| single | 单选题 | A/B/C/D |
| multiple | 多选题 | AB/ACD 等 |
| judge | 判断题 | true/false |

#### 2.3.2 题目完整数据结构
```javascript
Question {
  id: string,              // 题目ID
  type: 'single' | 'multiple' | 'judge',
  content: string,        // 题目内容
  options: [
    { key: 'A', value: '选项1' },
    { key: 'B', value: '选项2' },
    { key: 'C', value: '选项3' },
    { key: 'D', value: '选项4' }
  ],
  answer: string | string[], // 正确答案
  explanation: string,     // 答案解析（支持Markdown）
  videoUrl: string,       // 视频讲解链接（可选）
  images: string[],       // 配图（可选）
  theme: string,          // 所属主题
  tags: string[],         // 标签（难度、知识点等）
  difficulty: 'easy' | 'medium' | 'hard',
  stat: {
    playCount: number,   // 被使用次数
    correctRate: number   // 正确率
  },
  status: 'enabled' | 'disabled' | 'pending',
  createdBy: string,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

#### 2.3.3 主题数据结构
```javascript
Theme {
  id: string,
  name: string,           // 主题名称
  icon: string,           // 图标
  description: string,    // 主题描述
  coverImage: string,     // 封面图
  ageRange: '6-12' | '12-18' | 'all',
  questionCount: number,  // 题目数量
  color: string,          // 主题色
  sortOrder: number,      // 排序
  status: 'enabled' | 'disabled',
  createdAt: timestamp
}
```

### 2.4 游戏模块

#### 2.4.1 游戏结果数据结构
```javascript
GameResult {
  id: string,
  roomId: string,
  theme: string,
  playerResults: [
    {
      playerId: string,
      nickname: string,
      finalScore: number,
      correctCount: number,
      wrongCount: number,
      answerDetails: [
        {
          questionId: string,
          isCorrect: boolean,
          answerTime: number  // 答题用时
        }
      ]
    }
  ],
  startedAt: timestamp,
  endedAt: timestamp,
  duration: number        // 游戏时长（秒）
}
```

#### 2.4.2 计分规则
| 情况 | 分数 | 说明 |
|------|------|------|
| 答对 | +1 | 答对得分 |
| 答错 | -1 | 答错扣分（最低0分） |
| 超时未答 | 0 | 不扣分也不算对 |
| 抢答超时 | - | 自动跳过本题 |

### 2.5 后台管理模块

#### 2.5.1 题目管理
| 功能 | 描述 |
|------|------|
| 题目列表 | 支持分页、搜索、筛选（主题/难度/状态） |
| 添加题目 | 单条录入，支持富文本 |
| 编辑题目 | 修改任意字段 |
| 批量操作 | 批量导入/导出/启用/禁用 |
| 题目审核 | 待审核 → 通过/驳回 |

#### 2.5.2 主题管理
| 功能 | 描述 |
|------|------|
| 主题列表 | 展示所有主题及题目数量 |
| 创建主题 | 名称、图标、封面、描述 |
| 编辑主题 | 修改主题信息 |
| 题目关联 | 添加/移除该主题下的题目 |
| 主题排序 | 拖拽调整显示顺序 |

#### 2.5.3 系统配置
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 默认答题时间 | 新建房间的默认时间 | 15秒 |
| 默认题目数量 | 新建房间的默认题数 | 10题 |
| 是否启用注册 | 控制游客/强制登录 | 关闭 |
| 维护模式 | 临时关闭服务 | 关闭 |

## 四、API接口设计

### 4.1 房间相关
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/room/create | 创建房间 |
| GET | /api/room/:id | 获取房间信息 |
| POST | /api/room/:id/join | 加入房间 |
| POST | /api/room/:id/leave | 离开房间 |
| POST | /api/room/:id/start | 开始游戏 |
| POST | /api/room/:id/ready | 准备完成 |

| GET | /api/room/:id/result | 获取游戏结果 |

### 4.3 题库相关（管理端）

| DELETE | /api/questions/:id | 删除题目 |

### 4.4 主题相关（管理端）
| 方法 | 路径 | 描述 |
|------|------|------|

## 五、技术选型

### 5.1 前端
- **框架**：Vue 3 + Vite
- **状态管理**：Pinia
- **样式**：TailwindCSS
- **实时通信**：WebSocket

### 5.2 后端
- **运行时**：Node.js
- **框架**：Express
- **数据库**：SQLite（轻量，支持MVP）
- **实时通信**：Socket.IO

### 5.3 MVP技术栈
考虑到MVP快速验证，使用单文件HTML+JavaScript实现：
- 纯前端实现，数据存储在LocalStorage
- 使用BroadcastChannel实现同浏览器多标签页通信
- 支持模拟多人（同一浏览器多窗口）
